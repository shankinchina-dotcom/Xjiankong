## 一、目标与非目标

### 1.1 目标

将日报从「深色卡片列表」改成可连续阅读的 **深空研判报告**：用户打开首页后先得到一个明确结论，再按 8 个固定板块理解理由，最后才按需查看证据流与历史版本。

目标体验：

1. 首屏 10 秒知道当前版本、是否最新和今日一句判断。
2. 3 分钟连续读完 8 板块，不被重复卡片边框打断。
3. 需要核查时能从 `[Rxxx]` 跳到来源；链接修复前不得伪造“可回溯”。
4. 历史版本平时不占正文宽度，打开时才从右侧抽屉出现。

### 1.2 非目标

- 不改变 v2-alpha 的 8 板块名称、顺序或研判语义。
- 不增加信号强度、热度分、升温冷却、熵、簇图、昨日差分等没有上游真实数据的视觉化指标。
- 不新增图标库、前端框架或后端服务。
- 不将 RSS、热榜重新做成主页主角；它们仍是证据层。

## 二、推荐方向：深空研判报告

三个候选方向中，采用「深空研判报告」：它兼顾日报阅读、长期历史浏览和来源追溯；不采用仪表盘式“指挥台”，避免制造实时监控错觉；也不采用偏杂志的纯排版风格，避免削弱版本、来源与数据状态。

视觉关键词：**深空蓝、连续报告、低噪高密度、可追溯证据、克制发光**。

```text
顶部状态栏：产品名 / 当前版本 / 最新状态 / 历史入口
    ↓
今日总控台：一句判断 + 真实运行指标
    ↓
连续报告主体：8 个章节，而不是 8 张同质卡片
    ↓
证据流：低一级视觉权重的原始情报
    ↓
右侧历史抽屉：仅在用户主动打开时出现
```

## 三、信息层级与组件规格

### 3.1 顶部状态栏 `intel-topbar`

高度紧凑，承载全局上下文，不再与 Hero 做成两块视觉相同的大卡片。

| 元素 | 内容 | 规则 |
|---|---|---|
| 品牌 | `AI 情报终端` + 卫星线性图标 | 仅一个高权重品牌锚点 |
| 版本 | `2026-07-10 20:00` | 来自真实报告生成时间 |
| 状态 | `最新` 或 `历史版本` | 低噪 badge，不用红色 |
| 计数 | `今日 N 版` | 只在 manifest 已提供版本数时显示 |
| 操作 | `历史` 按钮 | 唯一主操作，打开抽屉 |

桌面端同行排列；窄屏下版本与状态换到第二行，历史按钮始终保留可点击面积。

### 3.2 今日总控台 `judgment-hero`

Hero 是唯一允许有明显蓝色能量渐变的区块。它不做仪表盘，而是报告封面：

1. eyebrow：`AI DAILY INTELLIGENCE / 当前版`，小字号等宽字。
2. 主标题：`今日一句判断`。
3. 结论：最多 3 行，优先呈现 `daily_judgment`；为空时显示真实空态，不填示例文案。
4. 指标行：热榜命中、RSS 命中、AI 8 板块、生成时间、今日版本数。所有值必须来自现有输出或未来 `history.json`。

Hero 不放大数字、不加动态图表、不把采集成功率包装成业务“评分”。

### 3.3 连续报告主体 `report-shell`

8 板块装入一个报告容器。容器本身使用一层弱边界与轻投影，章节之间用发光程度很低的分隔线和留白区分，避免出现 8 张“同一款卡片”的疲劳感。

每章 `report-section` 固定为：

```text
02  [line-icon] 研究／模型突破
    一句引导总结
    ┃ 判断条目；[R002] [R003]
    ┃ 判断条目；[R004]
```

- 两位编号是阅读锚点；编号色使用强调蓝，不用彩色状态含义。
- 左侧 `section-rail` 是 2px 蓝色细光轨；当前章节 hover 时仅提亮边框和轨道。
- `sec-summary` 是低饱和次级文字，来自 AI 生成结果；缺失时不显示空框。
- 条目为连续列表，单条不再套独立卡片；内容多时维持 4–6 条，不因视觉缩成两三条。
- 08 原始来源索引按来源类别分组，统计行只用真实可得数量。

### 3.4 来源引用与 404 保护 `source-ref`

`[Rxxx]` 是可点击的来源标记，而不是普通正文。正式接入前必须先完成“原始消息链接 404”只读诊断。

| 状态 | 呈现 | 行为 |
|---|---|---|
| 已验证 canonical URL | 蓝灰边框小标记 | 新窗口打开原始来源 |
| X/Nitter 有可用 fallback | 标记后展示来源名 | 弹出或展开原文 / 镜像两个明确入口 |
| 无可用链接或诊断未完成 | 中性不可点击文字 | 不显示伪链接，不将用户送往已知 404 |

AI 不生成 URL；链接只来自 SourceRef、RSS 原始 canonical URL 或经 Gate 3 验证的 URL 转换规则。

### 3.5 证据流 `evidence-stream`

证据流置于报告主体之后，视觉比研判正文弱一级：深色平面条目、低饱和细边、无厚阴影。

- 每条仅显示来源、时间、低噪 `NEW` badge 和标题。
- 标题三行截断；鼠标悬停只提升边框亮度。
- PC 使用 2 列自适应网格；移动端单列。
- `NEW` 使用琥珀或浅蓝胶囊，不能用纯红文字或无解释的红色数字。

### 3.6 历史抽屉 `history-drawer`

历史不常驻，不压缩报告正文。点击顶部历史按钮后从右侧滑出，约 `280ms`：

```text
当前版本（日期 / 时间 / 最新）
今日时间线（倒序多版）
近期（按日显示版本数与最近时间）
历史归档（可滚动）
```

关闭支持遮罩、关闭按钮和 `ESC`。打开历史版本时，正文顶部必须出现“当前查看：某时刻版本；这不是最新版本；查看最新”的可见提示。

## 四、视觉语言

### 4.1 色彩 token

| token | 值 | 用途 |
|---|---|---|
| `--bg-base` | `#07111f` | 深空底色 |
| `--bg-field` | `#0c1c33` | 顶部径向信号场 |
| `--surface-report` | `rgba(17, 39, 70, .82)` | 连续报告主体 |
| `--surface-evidence` | `rgba(12, 29, 52, .76)` | 证据流 |
| `--surface-focus` | `linear-gradient(135deg, rgba(27, 86, 166, .58), rgba(21, 52, 106, .82))` | Hero / 当前版本 |
| `--line-subtle` | `rgba(153, 190, 245, .18)` | 常规边界 |
| `--line-active` | `rgba(111, 178, 255, .55)` | hover / 当前状态 |
| `--text-primary` | `#edf5ff` | 主文字 |
| `--text-secondary` | `#9eb4d3` | 摘要、元信息 |
| `--accent-blue` | `#75b2ff` | 编号、链接、图标 |
| `--accent-cyan` | `#a5d4ff` | 小范围高亮 |

背景使用一个低对比径向场，而不是滚动变色动画。亮区只服务于 Hero、当前历史版本和 hover，不让全页都发光。

### 4.2 图标系统

取消彩色 emoji，改为本地内联 SVG 线性图标：`viewBox="0 0 24 24"`、`stroke="currentColor"`、`stroke-width="1.7"`，默认 `16px`，Hero 可用 `20px`。

不引入第三方图标依赖。图标只传达分区语义，标题文字仍是信息主角。

| 模块 | icon id |
|---|---|
| 今日核心判断 | `crosshair` |
| 研究／模型突破 | `flask` |
| 产品与工具变化 | `wrench` |
| 开源项目与开发者生态 | `network` |
| 大厂／融资／政策／算力动态 | `cpu` |
| X 关键人物观点 | `message-circle` |
| 风险与下周关注 | `triangle-alert` |
| 原始来源索引 | `book-open` |
| 证据流 / 顶部品牌 | `satellite` |
| 历史 | `history` |

### 4.3 排版与密度

- 字体：系统字体栈，正文优先 `PingFang SC`；数字和时间可用系统等宽字体。
- 正文行高 `1.7` 左右；报告正文最大行宽 `760–820px`，避免大屏一行过长。
- 页面容器最大 `1180px`；报告主体在桌面端保留足够左右留白。
- 大标题 28–32px；章节标题 18–20px；摘要 13–14px；来源标记 12px。
- 阴影仅使用 `0 10px 30px rgba(0, 0, 0, .16)` 以下的轻量级别。

## 五、响应式与可用性

| 视口 | 规则 |
|---|---|
| `≥ 1024px` | 报告全宽；证据流两列；历史抽屉 380px |
| `640–1023px` | 内容左右留白收缩；证据流仍可两列；抽屉最大 88vw |
| `< 640px` | 顶栏换行；Hero 指标自动折行；报告章节缩小内边距；证据流单列；抽屉占 92vw |

- 所有按钮和引用链接需要明确 hover/focus 状态；键盘 `ESC` 必须关闭抽屉。
- 颜色不是唯一状态表达：最新/历史/不可点击来源必须有文字。
- 长标题必须截断而非撑破容器；来源统计与空态必须可读。

## 六、数据与空态边界

| 数据缺失 | 页面行为 |
|---|---|
| 没有 `daily_judgment` | Hero 显示“本版未生成 AI 研判”，不填 mock 文案 |
| 某研判板块为空 | 保留标题与简短“暂无高置信信号”说明 |
| 尚无 `history.json` | 历史按钮可隐藏或显示“历史归档准备中”；不可展示硬编码历史时间线 |
| 来源 URL 未验证 | 展示来源 ID，不输出可点击链接 |
| RSS 为 0 | 证据流显示真实空态，不显示示例卡片 |

## 七、实现边界与验证要求

实现必须按既定闸门推进，当前不会直接进入代码：

1. **Gate 3：原始消息链接 404 只读诊断**，已完成；详见主策划的 Gate 3 审计结果。
2. **Gate 4：Codex 审计 mockup 与数据映射**，确认新视觉能由现有 AIAnalysisResult / SourceRef 和未来 history manifest 支撑。

实现前需确认相邻 TrendRadar fork 的工作树状态；不覆盖已有改动。生产同步、镜像构建、NAS 上传、容器重启、付费 AI 验证均另起闸门并由老板单独确认。

## 八、验收标准

- [ ] 页面首屏清楚区分状态栏、今日结论、真实指标；没有两块相似的大卡片。
- [ ] 8 板块视觉上是一份连续报告，不是同质卡片墙。
- [ ] 全站使用统一单色线性 SVG 图标，页面不残留彩色 emoji 作为系统图标。
- [ ] Hero、当前版本与普通区域存在清晰但克制的蓝色层级。
- [ ] RSS/热榜作为证据层，视觉权重低于研判正文。
- [ ] 历史抽屉默认零占用，支持遮罩、关闭按钮和 `ESC`。
- [ ] 所有数值、时间、版本、来源链接均可追溯到实际数据；缺失时显示真实空态。
- [ ] 来源链接不再把用户普遍导向已知 404；Gate 3 的诊断结论被落实。

## 九、交接给执行 Agent 的一句话
