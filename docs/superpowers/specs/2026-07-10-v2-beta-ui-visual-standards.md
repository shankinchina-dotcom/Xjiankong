## 一、权威硬约束（来自策划原文，不可违反）

以下直接决定「什么不能做」，违反任一条都算越界：

1. **只展示真实数据，不伪造趋势**（原文 §3.3）：
   - 可展示：报告生成时间、当日版本数、每版路径、热榜命中、RSS 命中、AI 分析是否存在、当前是否最新。
   - **禁止展示**：信号强度分数、熵值、升温/冷却判断、话题簇图、昨日差分。这些需要上游数据，不能靠 UI 编。
2. **不改 v2-alpha 的 8 板块框架**（原文 §3.4）：历史浏览和新 UI 是外壳升级，8 板块名称与顺序固定：
   1. 今日核心判断
   2. 研究/模型突破
   3. 产品与工具变化
   4. 开源项目与开发者生态
   5. 大厂/融资/政策/算力动态
   6. X 关键人物观点
   7. 风险与下周关注
   8. 原始来源索引
3. **视觉禁做项**（原文 §8.2 / §11）：
   - 不做大面积纯白卡片网格
   - 不做大面积紫色顶部横幅
   - 不做 RSS 单独绿色主题（RSS 须融入蓝色体系）
   - 不做红色圆点数字或不解释的热度数字
   - 不假装有实时仪表盘的复杂数据可视化
   - 不做话题簇图、信号强度、熵值、昨日差分
4. **历史浏览是受控页面，不是目录列表**（原文 §3.2 / §9.2）：不开启 Nginx `/html/` 目录列表；历史通过 `history.json` manifest + 受控页面呈现。

## 二、视觉系统标准（已确认）

### 2.1 整体基调：深色情报终端 + 淡饱和蓝渐变

采用策划原文 §8.1 推荐方向。色彩 token（mockup 落地值，可直接用）：

| 用途 | 值 |
|---|---|
| 页面背景（深色基底，要有存在感） | `#08111f` / `#0b1628`（径向或线性低饱和渐变，非纯黑、非寡淡浅灰） |
| 主卡背景 | `linear-gradient(135deg, rgba(32,87,160,.22), rgba(72,56,170,.16))` |
| 边框 | `rgba(150,190,255,.20)` |
| 主文字 | `#e8f1ff` |
| 次级文字 | `#93a8c8` |
| 强调蓝 | `#66a6ff` |
| badge 底色 | `rgba(102,166,255,.14)` |

**关键反差要求（老板明确反馈）**：背景与卡片之间必须有可辨识的反差，不能「寡淡同色系微差」。落地方式：深色基底 + 淡饱和蓝渐变卡片浮于其上，卡片用极轻投影（如 `0 2px 12px rgba(20,40,80,.25)`）与背景分离。**不要**回到 v2-alpha 那种接近纯白的浅灰底 + 白卡。

### 2.2 图标化标注（单色线性 SVG，贯穿全站）

- **每个研判板块标题都配统一的单色线性 SVG 图标**，不用彩色 emoji。图标使用 `stroke="currentColor"`、`stroke-width="1.7"`、`viewBox="0 0 24 24"`，默认 16px；不引入第三方图标依赖。
- 已确认映射：`crosshair` 今日核心判断、`flask` 研究/模型突破、`wrench` 产品与工具变化、`network` 开源项目与开发者生态、`cpu` 大厂/融资/政策/算力、`message-circle` X 关键人物观点、`triangle-alert` 风险与下周关注、`book-open` 原始来源索引。
- 全站模块对应 `satellite`（品牌/证据流）、`history`（历史）、`map-pin`（当前版本）、`clock-3`（今日时间线）、`calendar-days`（近期）。
- 图标是标注而非装饰：保持低饱和蓝色，标题文字仍是信息主角。

### 2.3 分散式一句话总结（老板强偏好）

- **每个板块标题下、详情列表前，放一句低饱和引导总结**（`.sec-summary` 风格），不是只在首屏放一段。
- 首屏 hero 仍保留一句全局总览（红色框那类总结的位置），但各板块也要有自己的总结句。
- 总结文案由 DeepSeek 在生成阶段产出（见原文 §12.3 分工），前端只负责排版呈现。mockup 阶段的总结是示意文案。

### 2.4 卡片与条目

- 研判板块：全宽渲染，并放入一个连续的报告主体；章节以编号（01/02/03…）、线性图标、左侧蓝色细光轨和分隔留白区分，不再把每一板块做成同质渐变卡片。每条判断带 `[Rxxx]` 来源引用标记（如 `[R001. source.id]`）。
- 证据流（RSS/来源卡片）：低饱和蓝卡，超长标题用 `-webkit-line-clamp: 3` 截断，NEW 用低噪胶囊（见 2.5）。
- hover 轻量提亮，不用大阴影（延续 Gate 2 标准）。

### 2.5 NEW / 状态标签

- **独立 class（如 `.rss-new-badge`），不复用 `.rss-author`，不用 inline style 红色**。
- 低饱和小胶囊：琥珀/淡蓝底色 + 低饱和文字，如 `background: rgba(245,158,11,.14); color: #b45309`（暗色下提亮）。**不要纯红大字**。

## 三、布局标准（已确认）

### 3.1 主页：单栏全宽，研判为主体

- 桌面端**不采用**左右分栏常驻侧栏。主内容区单栏全宽，AI 研判报告占满整屏。
- 顶部情报栏：项目名 / 当前版本时间 / `最新` badge / `今日 N 版` / `history` 线性图标按钮。
- 首屏 hero：`crosshair` 线性图标 + 一句话总览 + 真实指标（热榜 M/N · RSS X/Y · AI 8 板块 · 生成时间 · 今日版本数）。
- 下方 8 板块作为连续报告逐条渲染（单色线性图标 + 编号 + 分散总结 + 详细列表 + [R] 引用）。
- 再下方 RSS/证据流卡片区。

### 3.2 历史查看：右侧滑出抽屉（老板最终选择）

> 演化：常驻右侧窄栏（占版面）→ 底部折叠归档（仍占首屏）→ **抽屉式（最终确认）**。

- 平时零占用。点顶部 `history` 线性图标按钮从右侧滑出抽屉（约 0.28s 动画）。
- 抽屉内容：`map-pin` 当前版本 + `clock-3` 今日时间线（多版倒序）+ `calendar-days` 近期（逐日）+ AI 情报历史归档（逐日版本行，可滚动）。
- 关闭方式：点遮罩 / 点 `×` / 按 `ESC`。
- 移动端：历史入口改为顶部按钮触发抽屉，不遮挡正文。
- 视觉：当前版本高亮边框；最新版本低噪 badge；历史版本淡蓝渐变底，与研判区统一。

### 3.3 旧版本提示

打开非最新版本时，报告顶部显示（原文 §5.3）：
```
当前查看：2026-07-09 16:32 版本
这不是最新版本 · 查看最新
```

## 四、内容密度标准（老板反馈：要更丰富）

- 每个研判板块 **4–6 条**详细判断（不要 2–3 条薄卡片）。
- 板块 08 来源索引用**分类展示**（X / 媒体 / 论文开源 / 监管），并附统计行（如 33 源 · 19 条 [R] · 2 条 [H]）。
- RSS/证据流 **≥12 条**，多来源账号，含超长标题（测 3 行截断）与 NEW 低噪胶囊。
- 历史时间线补全当日多版（建议 6 版，如 20:00/18:30/16:32/14:10/12:03/08:02/04:01）。
- 近期/归档展 3–4 天。

> 高密度下必须保证层级清晰：图标 + 编号 + 分散总结 + 详细列表 + [R] 引用 的视觉权重递减有序。

## 五、交互标准

- 搜索：过滤后同步隐藏无可见条目的空分组（Gate 2 已实现逻辑，沿用）。
- 抽屉：见 3.2。
- 暗色为默认主题（与原文 §8.1 一致）；如做亮色切换需保证对比度，但 v2-beta 第一轮以暗色为主。
- 不引入需要后端服务的动态交互（原文 §11.9）。

## 九、已否决方向（避免重蹈）

| 方向 | 结论 | 原因 |
|---|---|---|
| 信号强度计 / 熵 / 升温冷却 / 昨日差分 | 否决 | 原文 §3.3 禁做，需上游数据不能编 |
| 话题簇图 | 否决 | 原文 §3.3 / §11 禁做 |
| 暖→绿滚动联动渐变背景 | 否决（探索过） | 老板最终选 spec 深色蓝渐变方向；复杂背景动画非必要 |
| 右侧常驻窄侧栏 | 否决 | 占版面，研判被挤（老板反馈「历史占比太多」） |
| 底部折叠归档区块 | 否决 | 仍占首屏；改为抽屉式零占用 |
| 大面积纯白卡片 / 紫色横幅 / RSS 绿主题 / 红色 NEW | 否决 | 原文 §8.2 明确不推荐 |
| 研判压成薄卡片（每板块一句话） | 否决 | 老板反馈「信息不够」，恢复全宽详细渲染 |
| 彩色 emoji 作为系统图标 | 否决 | 跨设备表现不一，削弱长期产品的科技感；改用本地单色线性 SVG |
| 8 张同质渐变卡片 | 否决 | 容易形成重复卡片墙；改为连续报告主体，用章节轨道与留白建立层级 |

## 十、给下一 agent 的一句话总结

做 v2-beta 前端，记住三条：① **深空蓝 Hero + 连续报告主体**（不要寡淡白底，也不要同质卡片墙）；② **单色线性图标贯穿全站 + 每板块一句分散总结**；③ **历史用右侧抽屉、平时零占用，研判全宽为主体**。所有指标只展示真实存在的，禁做项（信号强度/熵/簇图等）一律不碰。
